emdashii CRKBD Planned Updates

Working plan for bringing keyboards/crkbd/keymaps/emdashii up to the current TOTEM daily-driver shape, using the extra CRKBD keys for dedicated roles and adopting current QMK features. This revision reflects the decisions made in the planning session and is the build target.

Base · Nav+Symbol · Number · Function · Adjust No Gaming layer No Utility layer (combos stay on base) Home-row mods kept Layer Lock replaces Repeat

Locked-In Decisions

  • Caps Word moves to the left-middle pinky (the physical Caps Lock spot), as a plain CW_TOGG.
  • Repeat Key is dropped. The right-inner thumb is OSL(_FUNCTION) on the base layer and Layer Lock (QK_LLCK) on every other layer — one physical key with a sensible job on each layer.
  • Navigation is a momentary hold on the left-middle thumb (MO(_NAVIGATION)).
  • Number is a momentary hold on the lower-left pinky (MO(_NUMBER)), and its digits are mirrored onto the left hand so either hand alone can type numbers.
  • Word-delete (Ctrl+Backspace) moves off the old O+P combo onto the right-outer thumb (transparent on the non-base layers).
  • Layers: Symbols merge into Navigation (Nav+Symbol). Number and Function are their own layers. Adjust stays. Gaming and Utility layers are removed.
  • Single Nav key (left thumb only) — no second nav access on the right.
  • Combos remain on the base layer and the productivity macros are kept (see below).
  • OLEDs: off-hand half keeps the emdashii art; master half shows layer name, last key, WPM, and modifier/caps state.

⚠ Blocker: the current keymap will not compile

The checked-in keymap.c still uses keycodes that were removed from QMK. These must be replaced before anything else builds. This is the first implementation step.

QMK Features In Use

Right-inner thumb: OSL → Layer Lock

One physical key, layer-dependent. On base it is OSL(_FUNCTION) (tap → next key from Function → auto-return). On every other layer it is QK_LLCK (Layer Lock), which pins the layer you reached by holding so you can let go: hold Nav, tap this, release — now parked in Nav hands-free. Tap again to unlock.

References: One Shot Keys, Layer Lock.

Caps Word

CW_TOGG on the Caps Lock spot. Types one capitalized word/identifier then turns itself off at the next space. Cleaner than a real Caps Lock. Shift is still available on the home row.

Reference: Caps Word.

Combos & Macros

Combos stay on the base layer (no separate utility layer). Productivity actions remain firmware macros via SEND_STRING / process_record_user.

References: Combos, Send String.

RGB Matrix indicators

Per-key lighting via rgb_matrix_indicators_advanced_user() (still the current hook). Only the keycodes changed (RM_*); the indicator callback and rgb_matrix_set_color() are unchanged.

Reference: RGB Matrix.

WPM

Enabling WPM_ENABLE = yes exposes get_current_wpm() for the info OLED. Small flash cost; fine alongside LTO_ENABLE.

Reference: WPM.

Final Layer Structure

  1. Base: QWERTY, home-row mods, dedicated editing keys, dedicated layer keys.
  2. Navigation + Symbol: arrows + mouse on the right hand, symbols/brackets on the left. Momentary hold (left thumb).
  3. Number: numpad mirrored on both hands (right-hand digits duplicated on the left) plus math operators, so either hand alone can type numbers. Momentary hold (lower-left pinky).
  4. Function: F1–F12. One-shot access from the base-layer right-inner thumb.
  5. Adjust: boot, RGB, media/brightness, board settings. Toggle access from Nav/Number.
The architectural rule: layer keys are single-purpose. Holding always means “this layer,” tapping a letter always means text. No layer-tap on the normal editing path.

Consistent line-macro pattern: the three whole-line macros (SELLINE → COPY_LINE → DUPLINE) are adjacent pairs walking rightward along the bottom row (M+,, ,+., .+/), so they read as one family. EMDASH joins the home-row symbol/math combos at K+;, and DELEND (a partial-line trim) sits on its own cross-split G+H chord.

Removed: the old Q+W→Esc and O+P→word-delete combos (Esc is a dedicated key; word-delete is now the right-outer thumb). Note the 2024 combo changes (repress behavior, COMBO_MUST_* timing) for tuning. Reference: Combos.

Combo Map

Each link joins the two keys you press together; the badge in the middle is what it outputs. Use the buttons to isolate one family at a time — with everything on at once the home row gets busy.

Straight links are adjacent-key chords; arched links skip a key (e.g. J+L→~ and K+;→Em—). The two cross-split links (T+Y, G+H) join the inner index keys of the two halves.

Consistent line-macro implementation: the whole-line family is built as a strict superset chain — SELLINE = Home, Shift+End; COPY_LINE = SELLINE + Ctrl+C; DUPLINE = COPY_LINE + End, Enter, Ctrl+V. Same Home-anchored selection every time, so behavior is predictable regardless of cursor position.

Dropping

OS_SWAP (Mac/Win toggle), SNAP (screenshot), MAKE_H (compile command), and the EMAIL idea are not carried over. The old word-delete (Ctrl+Backspace) is no longer a combo — it lives on the right-outer thumb.

RGB Lighting Plan

Driven from rgb_matrix_indicators_advanced_user() in rgb.c. Base shows the home row plus the keys that jump to other layers. Each layer lights its most important keys and (where it makes sense) its navigation keys. Layers reached by holding (Nav, Number) do not show layer-jump hints — you are only there momentarily, so “how to get elsewhere” is noise. Toggle-reached Adjust keeps full hints.

OLED Plan

Off-hand half — art

Keep the existing raw-bitmap emdashii logo (oled_render_logo()), which is what emdashii_38 renders. No change beyond confirming it still draws after the refactor.

Master half — info

Four lines of live status:

  • Layer name — from oled_render_layer_state() (update the names to match the new 5-layer set; drop the Gaming case).
  • Last key pressed — reuse the existing keylog renderer.
  • Typing speed (WPM) — get_current_wpm() after enabling WPM.
  • Modifier / caps state — get_mods() / get_oneshot_mods() for S C A G, plus is_caps_word_on() and the caps-lock LED.
Required toggle: set WPM_ENABLE = yes in rules.mk (currently no). The broken/disabled set_keylog experiments in oled.c can be cleaned up; the r2g-derived keylog path already works and is what to build on.

Implementation Order

  1. Unblock the build: swap the removed keycodes (RESET→QK_BOOT, all RGB_*→RM_*) and confirm a clean compile of today's map.
  2. Trim layers: delete _GAMING; there is no separate utility layer to remove (combos already live on base). Update enum crkbd_layers and every reference.
  3. Rebuild the base layer per the map above: Tab/CapsWord/MO-Number pinky column, the new thumb cluster (Del / MO-Nav / Enter — LLock / Space / OSL-Fn), dedicated top-right Backspace.
  4. Merge Symbol into Navigation and keep Number, Function, Adjust as their own layers. Wire Adjust access + exits.
  5. Define the macros (kept + any chosen additions) and re-attach the base combos.
  6. Rewrite RGB indicators against the new key positions and the held-vs-toggle hint rule.
  7. OLED: enable WPM, fix layer names, add the WPM + mod/caps lines, keep the art on the off-hand.
  8. Build + flash after each step so regressions stay isolated.

Settled Details

  • Combo homes: EMDASH = K+; (grouped with the symbol/math combos); DELEND = G+H.
  • Number layer: digits mirrored on both hands; no extra Backspace/Tab on the left.
  • Right-outer thumb: word-delete on base, transparent on every other layer.
  • Right-mid pinky: stays the dedicated ' key — no second Enter. (', ", ` also live on Nav+Symbol.)
